# Agents Instructions

이 문서는 이 저장소에서 작업하는 에이전트/자동화 도구의 공통 실행 규칙입니다.

## 1) 브랜치/작업 시작 규칙

- `main`에서 직접 작업하지 않습니다.
- 항상 새 브랜치에서 시작합니다: `feat/<topic>`, `fix/<topic>`, `chore/<topic>`.
- PR 대상 브랜치는 항상 `main`입니다.

## 2) 구현/수정 원칙

- 요청 범위만 구현하고, 무관한 리팩토링은 하지 않습니다.
- 커밋은 기능 단위로 작게 나누고, 롤백 가능한 단위로 유지합니다.
- 사용자/로컬 실행 산출물(예: 임시 sqlite 파일, 로컬 노트/증적 파일)은 의도된 변경이 아니면 커밋하지 않습니다.

## 3) 검증 규칙

- 코드 변경 후 최소 검증 기준: `bun run ci`
- 머지 기준: CI 통과 + 커버리지 85% 이상
- 실패 시 원인과 복구 방법(롤백 포인트)을 PR 본문에 기록합니다.

## 4) PR 작성 규칙

- PR 제목 형식: `v<major>.<minor>.<patch> | <요약>`
- PR 본문은 `.github/PULL_REQUEST_TEMPLATE.md`를 그대로 따릅니다.
- 필수 섹션:
  - `## 개요`
  - `## 검증`
  - `## 핵심 체크`
  - `## 위험/롤백`
- `## 핵심 체크`의 체크박스 3개는 모두 `[x]` 상태여야 합니다.

## 5) 커밋 메시지 규칙

- 한글 커밋 메시지를 사용합니다.
- 권장 형식: `feat: ...`, `fix: ...`, `test: ...`, `chore: ...`, `revert: ...`
- 커밋 템플릿은 `.github/commit_template.md`를 사용합니다.

## 6) 참조 우선순위

- 1순위: `CONTRIBUTING.md`
- 2순위: `README.md`
- 3순위: `pm_notes/코드_규칙.md`

규칙 충돌 시 상위 문서를 우선합니다.
